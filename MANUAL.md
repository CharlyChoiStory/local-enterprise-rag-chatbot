## 1. 이 프로젝트는 무엇인가?

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                   │
│   💬  회사 내부 문서를 읽고                                       │
│       임직원 질문에 자동으로 답변해 주는                           │
│       AI 챗봇을 만들었습니다                                      │
│                                                                   │
│   📄  연차는 며칠 전까지 신청해야 해?                             │
│   🤖  → 최소 3영업일 전까지 HR포털에서 신청하셔야 합니다.         │
│         출처: [연차휴가규정.txt · 단락 3]                         │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 해결하는 문제

| 기존 방식 | 이 챗봇 방식 |
|---|---|
| 담당자에게 전화/메일 문의 | 챗봇에 바로 질문 |
| 여러 폴더를 뒤져서 문서 찾기 | 자연어로 질문하면 자동 검색 |
| 같은 질문 반복 응대 | 24시간 자동 응답 |
| 퍼블릭 AI에 회사 문서 업로드 | 회사 내부망에서만 동작 |

## 2. RAG란 무엇인가? — 쉬운 비유로 이해하기

### 📚 도서관 사서 비유

```
┌───────────────────────────────────────────────────────────────────┐
│                                                                     │
│   [일반 AI 챗봇 = 기억력 좋은 사람]                                │
│                                                                     │
│   ❓ 질문: "우리 회사 연차 규정이 어떻게 돼?"                      │
│   🤖 답변: "일반적으로 1년에 15일 정도..."  ← 인터넷 지식으로 추측 │
│            (우리 회사 규정을 모름!)                                 │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│   [RAG 챗봇 = 도서관 사서]                                         │
│                                                                     │
│   ❓ 질문: "우리 회사 연차 규정이 어떻게 돼?"                      │
│   📚 1단계: 회사 문서 창고에서 관련 문서를 찾는다                  │
│   📄 2단계: "연차휴가규정.txt" 3번 단락 발견!                      │
│   🤖 3단계: 그 내용을 바탕으로 정확한 답변 생성                    │
│   ✅ 답변: "최소 3영업일 전까지 HR포털에서 신청..."                 │
│            [출처: 연차휴가규정.txt · 단락 3]                        │
│                                                                     │
└───────────────────────────────────────────────────────────────────┘
```

### RAG = Retrieval-Augmented Generation

```
  R  →  Retrieval   =  검색    : 관련 문서 조각을 찾아오는 것
  A  →  Augmented   =  보강    : 찾은 내용으로 답변 재료를 보충하는 것
  G  →  Generation  =  생성    : AI가 최종 답변을 만들어 내는 것
```

### 🔄 RAG 작동 순서 (그림)

```
사용자 질문 입력
       │
       ▼
┌──────────────────┐
│   질문을          │
│   숫자(벡터)로    │  ← "연차 신청 기한" → [0.23, 0.87, 0.41, ...]
│   변환 (임베딩)   │
└──────────────────┘
       │
       ▼
┌──────────────────┐     ┌────────────────────────────────────┐
│   문서 창고에서   │ ←── │  📦 회사 문서 전체가               │
│   가장 비슷한     │     │     숫자(벡터)로 미리 저장된 DB    │
│   내용 Top-5 검색 │     │     (= Chroma Vector DB)           │
└──────────────────┘     └────────────────────────────────────┘
       │
       ▼
┌──────────────────┐
│   검색된 문서     │  ← 연차휴가규정.txt 단락 3
│   조각 5개를      │  ← 01_연차_휴가_운영_가이드.pdf page1
│   AI에게 전달     │  ← 재택근무운영지침.txt 단락 2  ...등
└──────────────────┘
       │
       ▼
┌──────────────────┐
│   로컬 AI(LLM)이  │  ← llama3.1:8b 모델 (내 PC에서 실행)
│   문서 내용 기반  │
│   답변 생성       │
└──────────────────┘
       │
       ▼
사용자에게 답변 + 출처 표시
```

### 🆚 전통 검색 vs RAG 비교

```
┌────────────────┬──────────────────────┬──────────────────────┐
│   구분          │   전통 키워드 검색    │   RAG 검색           │
├────────────────┼──────────────────────┼──────────────────────┤
│   검색 방식     │ 단어 일치            │ 의미/문맥 유사도      │
│   질문 예시     │ "연차 신청"          │ "휴가 얼마 전에 내야" │
│   결과          │ 파일 목록 반환       │ 자연어 답변 반환      │
│   출처 표시     │ 없음                 │ 파일명+위치 표시      │
│   정확도        │ 키워드 없으면 실패   │ 의미가 비슷하면 성공  │
└────────────────┴──────────────────────┴──────────────────────┘
```

## 3. 왜 기업 보안이 중요한가?

### 🚨 퍼블릭 AI 사용 시 발생하는 보안 위험

```
직원이 ChatGPT에 질문할 때 일어나는 일
                                        
직원 PC                인터넷             AI 회사 서버
   │                      │                    │
   │  "우리 회사 급여      │                    │
   │   체계가 어떻게..."   │                    │
   │──────────────────────┼───────────────────►│
   │                      │                    │ 📋 질문 내용 저장됨
   │                      │                    │ 🤖 AI 학습 데이터로
   │                      │                    │    활용될 수 있음
   │◄─────────────────────┼────────────────────│
   │  답변 수신            │                    │
                                        
⚠️  문제점:
   1. 회사 기밀 문서 내용이 외부 서버로 전송됨
   2. 개인정보(직원 급여, 인사정보)가 외부로 유출될 수 있음
   3. 영업기밀, 특허 관련 정보가 경쟁사에 노출될 수 있음
   4. GDPR, 개인정보보호법 위반 가능성
```

### 🏢 기업이 겪는 실제 사례

```
┌─────────────────────────────────────────────────────┐
│  📰 실제 발생한 사례들                               │
│                                                     │
│  • 삼성전자 (2023): 반도체 코드를 ChatGPT에 입력      │
│    → 기밀 유출 우려로 사내 ChatGPT 사용 금지         │
│                                                     │
│  • 미국 대형 로펌: 법률 문서를 AI에 업로드            │
│    → 의뢰인 개인정보 유출 문제로 소송 위기            │
│                                                     │
│  • 다수 금융기관: 고객 데이터 AI 입력 금지 내규 신설  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 4. 보안 원칙: 완전 내부 설치형(On-Premise) 개발

### 🔒 이 시스템의 보안 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│                   🏢 회사 내부 네트워크                          │
│                                                                   │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐   ┌──────────┐  │
│   │ 직원 PC  │───►│Streamlit │───►│ Chroma   │   │  Ollama  │  │
│   │(브라우저)│    │  웹 UI   │    │Vector DB │   │로컬 LLM  │  │
│   └──────────┘    └──────────┘    │(로컬 저장)│   │(로컬 실행)│  │
│                        │          └──────────┘   └──────────┘  │
│                        │                │              │         │
│                   ┌────▼──────────────────────────────┐         │
│                   │          Python 애플리케이션        │         │
│                   │  parsers → chunker → embeddings   │         │
│                   │  → vector_store → rag → llm       │         │
│                   └───────────────────────────────────┘         │
│                                    │                             │
│                   ┌────────────────▼─────────────────┐          │
│                   │       📁 회사 문서 폴더            │          │
│                   │  data/source_docs/               │          │
│                   │  ├── hr/      (인사/연차)         │          │
│                   │  ├── finance/ (출장비/경비)        │          │
│                   │  ├── welfare/ (복지/경조사)        │          │
│                   │  ├── it/      (IT장비/보안)        │          │
│                   │  └── policy/  (내규/매뉴얼)        │          │
│                   └──────────────────────────────────┘          │
│                                                                   │
│   ════════════════════════════════════════════════════           │
│   🚫  인터넷 구간으로 나가는 데이터: 없음                          │
│   🚫  외부 AI API 호출: 없음 (OpenAI/Anthropic/Google 금지)       │
│   ✅  모든 AI 처리: 회사 내부 서버에서만 실행                      │
└─────────────────────────────────────────────────────────────────┘
```

### ✅ 적용된 보안 규칙 체크리스트

```
보안 항목                              적용 여부    방법
─────────────────────────────────────────────────────────────
퍼블릭 AI API 호출 금지                ✅ 적용     Ollama 로컬 모델 사용
외부 망으로 문서 전송 금지              ✅ 적용     모든 처리 로컬 실행
API Key 관리 (외부 AI 항목 없음)        ✅ 적용     .env에 외부 AI key 없음
회사 문서 로컬 보관                    ✅ 적용     data/source_docs/ 에만 저장
Vector DB 로컬 보관                    ✅ 적용     data/chroma/ 에만 저장
로그에 원문 전체 저장 금지              ✅ 적용     파일명·chunk ID·점수만 기록
.env 파일 Git 업로드 금지              ✅ 적용     .gitignore에 .env 포함
```

### 🆚 퍼블릭 AI vs 이 시스템 비교

```
┌─────────────────────┬──────────────────────┬──────────────────────┐
│   항목               │   ChatGPT 등 퍼블릭   │   이 시스템 (로컬)    │
├─────────────────────┼──────────────────────┼──────────────────────┤
│   데이터 이동        │ 인터넷 경유 (위험)    │ 내부망만 (안전)       │
│   문서 기밀성        │ 보장 안 됨            │ 완전 보장             │
│   사용 비용          │ 월정액/종량제 유료    │ 무료 (오픈소스)       │
│   인터넷 필요        │ 필수                 │ 불필요                │
│   커스터마이징       │ 제한적               │ 완전 자유             │
│   규정 준수(컴플라이언스)│ 어려움             │ 쉬움                  │
└─────────────────────┴──────────────────────┴──────────────────────┘
```

#### 🔍 2단계: 문서 파서(Parser) 구현

```
파일 확장자    사용 라이브러리    추출 단위    반환 형태
──────────────────────────────────────────────────────
.pdf          PyMuPDF           페이지별     page 1, page 2...

──────────────────────────────────────────────────────

parse_document() 반환 구조:

{
  "document_id": "해시값 16자리",     ← 파일 고유번호 (중복방지)
  "filename":    "연차휴가규정.txt",  ← 파일명
  "file_type":   "txt",             ← 파일 종류
  "category":    "hr",              ← 카테고리 (폴더명 자동감지)
  "sections": [                     ← 섹션 목록
    {"location": "paragraph 1",
     "text": "연차휴가는 1년 이상..."}
  ],
  "error": null                     ← 파싱 실패 시 에러 내용
}
```

#### ✂️ 3단계: 청킹(Chunking) 구현

```
청킹이란? 긴 문서를 AI가 소화할 수 있는 크기로 조각내는 것

원본 문서 (매우 긴 텍스트)
┌────────────────────────────────────────────────────┐
│연차휴가 규정 1조: 1년 이상 근속한 직원에게...       │
│2조: 연차 신청은 3영업일 전까지...                  │
│3조: 잔여일수는 HR포털에서...                        │
│... (계속) ...                                      │
└────────────────────────────────────────────────────┘
                     │
                     ▼  1600자씩 자르기 (250자 겹침)
                     
chunk 0        chunk 1        chunk 2
┌──────────┐  ┌──────────┐  ┌──────────┐
│1조: 1년  │  │2조: 연차 │  │3조: 잔여 │
│이상 근속 │  │신청은 3  │  │일수는 HR │
│한 직원에 │  │영업일 전 │  │포털에서  │
│게...     │  │까지...   │  │...       │
│──────────│  │──────────│  │──────────│
│←1600자→  │  │←1600자→  │  │←1600자→  │
│    ↑250자 겹침↑         │           │
└──────────┘  └──────────┘  └──────────┘

겹침(overlap) 이유: 문단 경계에서 내용이 잘리지 않도록
```

#### 🔢 4단계: 임베딩(Embedding) + Chroma DB 저장

```
임베딩이란? 텍스트를 AI가 계산할 수 있는 숫자 목록으로 변환하는 것

"연차는 3영업일 전까지"  →  [0.23, 0.87, 0.41, 0.92, ...]
"휴가 신청 마감일"       →  [0.21, 0.85, 0.44, 0.90, ...]  ← 비슷한 숫자!
"오늘 점심 뭐 먹지?"    →  [0.91, 0.12, 0.73, 0.05, ...]  ← 다른 숫자


저장 과정:

67개 chunk
    │
    │  bge-m3 모델로 각 chunk를 숫자로 변환
    ▼
┌────────────────────────────────────────────┐
│           Chroma Vector DB                 │
│                                            │
│  ID              │ 벡터              │ 원문  │
│  ────────────────┼───────────────────┼───── │
│  abc123_0        │ [0.23, 0.87, ...] │ "연차"│
│  abc123_1        │ [0.45, 0.23, ...] │ "출장"│
│  def456_0        │ [0.91, 0.33, ...] │ "복지"│
│  ...             │ ...               │ ...  │
└────────────────────────────────────────────┘

결과: 67개 chunk → 23초 만에 저장 완료 ✅
```

## 📋 용어 정리

```
용어              설명
──────────────────────────────────────────────────────────────────
RAG               Retrieval-Augmented Generation의 약자.
                  "검색-보강-생성"의 세 단계로 동작하는 AI 기술

LLM               Large Language Model. 대형 언어 모델.
                  ChatGPT 같은 대화형 AI의 기반 기술

임베딩            텍스트를 숫자 벡터로 변환하는 기술.
(Embedding)       의미가 비슷한 문장은 비슷한 숫자를 가짐

벡터 DB           숫자(벡터)를 저장하고 유사도로 검색하는 DB.
(Vector Database) 이 프로젝트에서는 Chroma 사용

Ollama            오픈소스 로컬 AI 실행 플랫폼.
                  인터넷 없이 PC에서 LLM을 실행할 수 있게 해줌

Streamlit         Python으로 웹 앱을 빠르게 만드는 프레임워크

청크(Chunk)       긴 문서를 작게 조각낸 단위. 1600자 기준

바이브코딩        자연어로 AI에게 지시하여 코드를 생성하는 개발 방식
(Vibe Coding)

On-Premise        인터넷 없이 자체 서버/PC에서만 운영하는 방식
(온프레미스)

POC               Proof of Concept. 개념 검증용 시범 구현
──────────────────────────────────────────────────────────────────
```
